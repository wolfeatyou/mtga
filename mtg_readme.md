# MTG Arena Draft Helper — инструкция по запуску

Помощник для **MTG Arena**: живой разбор пака во время драфта (главное) + опциональный
наблюдатель за матчем (медленный, можно не включать).

Работает с двумя сетами:
- `sos` — Secrets of Strixhaven (Premier Draft) — основной
- `mkm` — Murders at Karlov Manor (Quick Draft)

---

## 0. Разовая настройка (уже сделано, проверь если не работает)

1. **Arena → Settings → Account → Detailed Logs (Plugin Support)** → ВКЛ
2. **Перезапусти Arena** после включения (без рестарта логов не будет).
3. Логи лежат в: `~/Library/Logs/Wizards Of The Coast/MTGA/Player.log` (+ `Player-prev.log`)
4. Python 3 + (опционально для AI-советов) `pip3 install anthropic`

## 1. Как включить помощника в новой сессии Claude Code

Открой Claude Code в папке `/Users/wolfeatyou/claude/uixta` и скажи примерно так:

> «Помоги с драфтом SOS. Я открываю пак в Arena, ты читаешь лог и говоришь какую карту брать.
>  Запускай `python3 draft_live.py sos` после каждого моего пика. Будь быстрым — одна строка: какую карту брать и почему.»

Claude будет:
1. Запускать `python3 draft_live.py sos` (читает текущий пак из лога)
2. Смотреть карты, отсортированные по силе (17Lands GIH WR)
3. Говорить какую брать с учётом твоего пула, цветов и кривой

**После каждого пика** проси: «pak» / «следующий» — Claude перечитает лог и оценит новый пак.

## 2. Команды драфт-помощника

```bash
python3 draft_live.py sos        # ⭐ МГНОВЕННЫЙ текущий пак (не блокируется) — для интерактивных пиков
python3 draft_live.py sos watch  # БЛОКИРУЮЩИЙ — ждёт НОВЫЙ пак (для фона/unattended; лагает в интерактиве)
python3 draft_live.py mkm        # то же для MKM (Quick Draft)
python3 draft_live.py sos raw    # отладка: сырые draft-строки лога (если пак не виден)
```

**⭐ СТАНДАРТНЫЙ ВОРКФЛОУ ДРАФТА (выбор игрока: авто + мгновенное чтение):**
1. Claude держит `draft_live.py sos watch` в фоне — он БУДИТ на новом паке.
2. Когда сработал — Claude НЕ советует по выводу watch (он может быть устаревшим). Вместо этого делает
   **мгновенное чтение `python3 draft_live.py sos`** (без watch) → видит ТЕКУЩИЙ пак → советует по нему.
3. Перезапускает watch на следующий пак.
4. **Игрок ждёт ~30 сек пик перед кликом** (из-за ~40с латентности Claude; быстрее — обгонит любой режим).

Почему: `watch` (push) показывает пак на момент срабатывания, а ответ Claude идёт ~40с → можно опоздать к уже
пикнутому паку. Мгновенное чтение (pull) всегда даёт актуальный пак. Слово «смотри» не обязательно — Claude сам
перечитывает текущий пак при срабатывании. (Тот же приём `once`, что у матча.)

Что показывает:
- **Текущий пак**: карты по убыванию GIH WR, с тиром `[A+/A/B+...]`, GIH%, ALSA (как поздно вертится)
- **Бустер N, пик M** — где ты в драфте
- **Твой пул**: все взятые карты, баланс цветов, кривая маны (скоуп по текущему draftId,
  склейка `Player-prev.log` + `Player.log`, дедуп по координате — переживает обрыв связи)

Если пак не находится — запусти `raw` и покажи вывод Claude, он подстроит парсер.

## 3. Принципы пика (Claude следует им)

1. **P1P1** — самая сильная карта (бомба/премиум-удаление), не женись на цвете рано.
2. **Удаление и evasion** недооценить трудно.
3. **Цель колоды:** ~17 земель / 16-17 существ / 6-7 не-существ (≈23 нонленда).
4. **При равном GIH — ТЕЛО важнее N-го реактива** (урок: перебор контрспеллов = мало угроз).
5. **Читай сигналы:** сильная карта цвета идёт поздно = цвет открыт.
6. **Мана-база:** при перекосе в double-pip (RR, UU) — больше источников этого цвета или фикс.
7. **При близком/равном пике ТЕЛО на кривой (особенно 2-дроп) > небордовый «движок»/билд-эраунд.** Игрок стабильно проигрывает go-wide агро из-за нехватки тел и дёрдла на небордовых картах (см. §6). Это **жёсткий фильтр на каждом пике**, не сноска: если в собственном разборе пика мелькает «небордовая/медленная/durdle» — это РЕШАЕТ против неё. И не подгоняй счёт существ: **сплеш-существа (без фикса) и Vehicle (нужен экипаж) НЕ считаются телами на кривой**. Реальный провал: взял Death to Our Enemies (энчант-движок) вместо Agents of HYDRA (2-дроп), насчитав «~10 существ» через сплеш+Quinjet — игрок поймал. Не строй метрику ради блестящей карты.

### SOS архетипы (пары школ)
- **Lorehold (W/R)** — flashback/инстант-сорсери + кладбище, агро-мидрейндж (сильнейшая пара)
- **Silverquill (W/B)** — триггеры от заклинаний, агро
- **Prismari (U/R)** — opus, spellslinger
- **Quandrix (U/G)** — +1/+1 счётчики, рамп, converge
- **Witherbloom (B/G)** — infusion + lifegain
- Мета: «white vs soup» — не-белые сваливаются в 4-5-цветные converge-супы

Подробности: `draft_cheat.md`, `sos_tier.md`, `mkm_tier.md`.

## 4. Наблюдатель за матчем (ОПЦИОНАЛЬНО — медленный)

> ⚠️ **Советы прилетают с задержкой ~30-40 сек** (Claude просыпается → читает лог → отвечает),
> поэтому в реальном бою часто бесполезно. Встроенный быстрый rule-based советник в скрипте
> работает мгновенно, но грубый. Если играешь сам — этот блок можно игнорировать.

```bash
python3 match_watch.py sos loop both          # ⭐ ЛЕНТА В ТЕРМИНАЛ (рекомендуется): совет на твой ход + на блоки в атаку соперника. Простой язык, без сленга. Свой Terminal, Ctrl+C — выход.
python3 match_watch.py sos loop myturn       # только твой ход
python3 match_watch.py sos watch            # = watch myturn (по умолчанию)
python3 match_watch.py sos watch myturn     # ОДИН сигнал в начале твоего хода (главная фаза 1)
python3 match_watch.py sos watch oppcombat  # на блок-степе соперника (для подсказок по блокам)
python3 match_watch.py sos watch all        # старое: каждый твой приоритет (спамит каждый шаг)
python3 match_watch.py sos once             # один снимок текущего состояния
python3 match_watch.py sos raw              # отладка GRE-формата
```

**Scope вотчера (договорённость с игроком):** по умолчанию `myturn` — подсказка **ровно один раз за ход, в начале ТВОЕГО хода** (Phase_Main1), а не на каждом приоритете. Это убирает спам (раньше срабатывало на main/атаке/блоке/каждом триггере). `oppcombat` добавляет подсказку на ход соперника (объявление блоков) — включать по запросу игрока. `all` = прежнее поведение.

- `watch` шлёт macOS-уведомление (баннер поверх Arena) при твоём ходе + быстрый совет по правилам.
- **`quick_advice` переписан под W/B Silverquill** (мгновенно, без API): читает жизни + ширину бордов и выдаёт режим — 🎯 поиск летала (опп низко: считай силу атаки, Together as One +2, памп на безответного) / СТАБИЛИЗИРУЙ (позади или их ширина: крупный блокёр + снос на жирное) / агрессор (равно/впереди: развивай+атакуй) — плюс дисциплина трюков (держи на бой) и предупреждение о мана-скрю. Баннер = быстрый тактический слой; Claude в чат даёт глобальную стратегию (медленнее).
- Снимок: ход/фаза/приоритет, жизни, рука, столы, стек, кладбище.
- Скоуп по последнему `GameStateType_Full` (только текущая игра, без мусора прошлых).
- Твоё место (seat) = первый `systemSeatIds` в логе.
- AI-совет: положи `ANTHROPIC_API_KEY=sk-...` в файл `.env` рядом со скриптом → второй баннер
  через 3-5 сек (модель haiku). Без ключа — только rule-based.

> ⚠️ Реал-тайм AI-совет в бою структурно не вытягивает темп (нет пауз) — для разбора игр
> используй пост-гейм (ниже), он без латентности.

## 4b. Пост-гейм разбор (`analyze_game.py`) — лучший ROI, без латентности

Разбирает игру из лога ПОСЛЕ партии: результат, стартовая рука + мулиган-решение,
ходы (что разыграно/умерло), финальный борд, динамика жизней.

```bash
python3 analyze_game.py        # последняя сыгранная игра
python3 analyze_game.py -2     # предыдущая (индекс игры с конца)
```

Под капотом (почему теперь надёжно — после серии багфиксов):
- **Границы игр** — по сбросу `gameStateId` на ≤3 (ресинки посреди партии поглощаются, не считаются играми).
- **Место (seat) — ПЕР-ГЕЙМ** (меняется play/draw; глобальный первый seat врёт в поздних играх — был баг).
- **Борд — по типу карты** (`cardTypes`): земли-жетоны не идут в борд, существа отдельно от Treasure/артефактов (раньше фильтр был по имени и считал земли-жетоны за борд — был баг).
- **Стартовая рука + Keep/Mulligan** — из опенинг-стейта (работает, если срез захватил 1-й ход).

## 6. Уроки по матчам (что помнить)

### Разбор: читай данные, не выдумывай (урок из коуч-провала)
0. **Способности ≠ заклинания.** Повторяющийся «каст спелла каждый ход» в старом выводе был
   **багом**: способность на стеке логируется с `grpId` = id СПОСОБНОСТИ. Фикс резолвит её через
   `GameObjectType_Ability` + `objectSourceGrpId` → строка `⚡ способность — <карта>` и блок
   **`=== ДВИЖКИ / ПОВТОРНЫЕ СПОСОБНОСТИ ===`**. **Читай этот блок первым:** перекос по жизням
   почти всегда от lifelink/боевого триггера (Startled Relic Sloth, Muse Seeker), а не от
   «загадочного спелла». **Не приписывай потерю жизней догадке** — сверься со строками `⚔`/`🛡`/⚡
   и блоком ДВИЖКИ. Если потеря не объясняется боем — ищи lifelink/drain-движок в сводке, а не сочиняй.
   (Реальный провал: списал поражение на «жадную гонку», хотя гонку игрок почти выиграл — добил
   его оппонентский lifelink-трампл Sloth, которого парсер прятал.)

### Опознай движок → примени ПРАВИЛЬНЫЙ ответ (урок из проигрыша Repartee-деке)
- **«Prepared»: чтобы включить условие — ДОНАТЬ ETB-токен оппоненту.**
  `Emeritus of Truce` ({1}{W}{W}) — НЕ adventure. ETB: «**target player** creates a 1/1 Inkling»,
  **затем, если у оппонента существ больше, чем у тебя → становится prepared**, и пока подготовлено,
  можно скастить **копию `Swords to Plowshares`** ({W}, **exile target creature**) со стола.
  Ключ: «target player» — **ты выбираешь, кто создаёт токен.** Дашь токен СЕБЕ → свой счёт +1 →
  условие НЕ срабатывает → prepared нет → exile'а нет. Дашь токен ОППОНЕНТУ → его счёт +1 →
  «у оппонента больше» выполняется → prepared → открывается Swords → **exile в движок** (тут —
  Inkshape Demonstrator 3/4, Ward {2}, Repartee +lifelink, ×8; каст = {W}+Ward {2} = 3 маны).
  Проигрыш: игрок отдал ETB-токен СЕБЕ (лог: `ты: на стол → жетон`), условие не сработало, Swords
  не появился, движок дрейнил 15→2. **Размен — подарить 1/1 флаер, чтобы заэкзайлить ×8 lifelink-мотор
  деки — отличный.** Неочевидно: донат собственного токена врагу ради включения своего prepared.
  **Алгоритм: ДВИЖКИ → опознать движок → проверить, не открывает ли мой ETB/Prepared exile (и какой
  таргет условия для этого нужен) → спустить exile на движок.**
  (Коуч-поправки ×2: сперва назвал Swords стороной adventure — это Prepared; потом решил, что Emeritus
  «стоял подготовленным» — нет, prepared не включился из-за выбора таргета токена. Реальную причину
  нашёл игрок.)
- **Знай, что твой «removal» реально делает (castability ≠ имя).** `Last Gasp` (−3/−3) НЕ убивает
  3/4 (Inkshape) — выживает 0/1. `Render Speechless` ({2}{W}{B}) — это **ДИСКАРД** (вскрыть руку +
  сбросить) **+ каунтеры**, а НЕ removal существа; кастить его РАНО, чтобы вырвать пейофф/ключевой
  спелл, а не держать до конца. (В проверке колоды он был ошибочно записан в «removal».)
- **Repartee-lifegain/дрейн — тяжёлый матчап:** ранний рейс почти выиграл (опп до 13), но
  lifelink+дрейн переворачивают гонку. Единственный план — **выключить движок exile'ом**, гонкой
  его не обогнать.

### WB Silverquill + общие (по разбору ~5 игр, счёт 2-3)
1. **Кривая решает — добирай низ (1-2 дропы).** КАЖДОЕ поражение WB — медленный старт
   (первое тело на 4-6 ходу) → отстал по борду. Самый сильный рычаг, и он на драфте.
2. **Go-wide — проигрышный матчап.** Точечное removal бессильно против токенов; твой ответ —
   **Moment of Reckoning** (свип до 4 целей). Береги под их широкий борд, не трать рано как одиночное removal.
3. **Со свипом в руке НЕ тапайся в атаку.** На низких жизнях держи блок, доживи до Moment,
   потом сноси и дави. (Проигрыш: атаковал вместо «блок + Moment следующим ходом» — а это была победа.)
4. **Против бёрна/темпо жизни — ресурс.** Не гони в их бёрн-радиус; evasion-угрозу убивай первой;
   на ровной гонке играй вокруг баунса/трюка — оставляй подушку (блокер + запас жизней).
5. **lifelink/Repartee-движок включай рано.** Наводи дешёвый спелл на своё Repartee-существо
   (Inkshape Demonstrator) → бьёшь и лечишься в один ход. В победах это и решало (23-4, 21-5).
6. **WWBB-бомба застревает на бело-тяжёлой базе.** Считай Moment «по правильным землям»,
   не рассчитывай на неё по кривой против быстрого агро (в проигрыше простояла до 14-го хода).

### Универсальные
7. **Считай цвета маны, а не только число земель.** Double-pip (RR/UU/BB) требует 2 источника.
8. **Не атакуй стенами** (0/4, 0/3) — они для блока.
9. **Читай кейворды существ соперника ПЕРЕД боем/removal.** (R/W Lorehold, проигрыш vs go-wide —
   два мисплея в одной игре:) (а) план «на затапанного» / дисконт Ajani's Response сорвался — у атакующего
   был **vigilance** → он не тапнулся; (б) заблокировал существо с **double strike** не заметив →
   блокер умер до ответки, размен слит. Перед коммитом смотри vigilance / double strike / deathtouch / trample.
10. **Flash-копи-комбо (Dualcaster) — по мане и по тайму.** Копию кидай ПОСЛЕ того, как целевое
    заклинание уже на стеке, и только если хватает маны на ВСЮ линию. Не коммить ману в комбо, которое не докинешь.
11. **vs go-wide без масс-removal — держи блокеры, не разменивай весь борд в атаку.** Потерял всю доску
    в их go-wide → остался на 5 без тел против ~10 существ, 2 точечных removal не спасли. Запас блокеров > лишней атаки.

### Историческое (ветка UR #1)
9. Не передерживай ответы (свип + широкий борд = кастуй сразу). Essence Scatter — для летунов
   (контра бьёт по стеку, не по телу). Artistic Process не бьёт в лицо. *(детали — в git/старых заметках)*

### Матч vs go-wide лайфгейн (B/G Witherbloom — разбор победы 22 хода, был скейр 18→7)
10. **Впереди по жизням против go-wide = ты КОНТРОЛЬ, а не агро.** Не тапайся в атаку, когда
    уже ведёшь по жизням и не имеешь ЧИСТОГО летала. Оставляй команду в блок — оппу самому надо идти в гонку.
11. **Считай их alpha с командным пампом ПЕРЕД своим тапаутом.** `Root Manipulation`/анфемы дают
    всей ораве +2/+2 + menace + лайфгейн-за-атакующего → широкий борд бьёт на 11 и откачивается. Скейр
    18→7 был именно от недосчитанной альфы после моего тапаут-аута.
12. **Инстант-интеракцию держи на ИХ боевой шаг, не лей в свою главную фазу.** `Dissection Practice`/

### Урок ПРО ИНСТРУМЕНТ (разбор драфта UR Prismari, draftId 72e2ba37) — почему советник ошибался
Разобрали драфт по полным пакам (что лежало vs что взял, 17Lands GIH). 3 ошибки/8 спорных из 42. Диагностика причин:
1. **Главная причина — системная, в самой логике `draft_live.py`.** Пак сортируется **только по глобальному GIH**, а единственная осведомлённость о пуле — `cast_flag`, который ШТРАФУЕТ карты вне цветов (`✗offcolor`/`~splash`). Это **оптимизатор «лучшая карта в ТЕКУЩИХ цветах»** без кросс-пиковой памяти. На P2p3 он буквально пометил бомбу Scolding Administrator (GIH 60.5, WB) как `✗offcolor`-ловушку, и советник пошёл на 49.7 в цвете — пока WB был распахнут (Scolding 60.5, Render Speechless 57.4, Cost of Brilliance вернулся p4→p12). Детектора сигналов/колеса/разрыва мощности в коде **не было**.
2. **Дыра в данных.** В `sos_set.json` не было **63 архив-карт** (весь Mystical Archive: Burst Lightning, Cyclonic Rift, Vampiric Tutor, Knockout Maneuver…) — у них был рейтинг, но не было цвета/пипов, поэтому флаги не считались, а оценки врали (зелёный Knockout Maneuver выглядел «бесцветным»).
3. **Латентность** (Opus медленный → пропуск пика, см. память) — усугубляет: пик-за-пиком берёшь верхнюю в-цвете и едешь.
4. **Парсер чист** — размеры паков (0/42 ошибок), координаты, GIH-покрытие в норме. Не он.

**Внесённые правки:**
- **Детектор сигналов** в `draft_live.py` → печатает блок `─── СИГНАЛЫ ───` ПЕРЕД паком: `⚑ СИЛЬНЕЕ ВНЕ ЦВЕТА` (разрыв ≥3 GIH, пики ≤9), `⚑ ПИВОТ?` (пики 6–10, ≥2 карт ≥56 вне цвета = полоса открыта), `⚑ КОЛЕСО` (под=8: пак с пика P−8 вернулся подмножеством, ≥54 GIH), `⚑ SOUP-AUDIT` (пик 5/6: счёт фикса → соус vs 2 цвета). Детали в `SKILL.md`.
- **Дозалив 63 архив-карт** в `sos_set.json` (бэкап `sos_set.json.bak`) — все рейтинговые карты теперь резолвятся с цветом/пипами/картинкой.

**Честная оценка эффекта (НЕ переоценивать):** дозалив данных — безусловный плюс (класс ложных оценок закрыт). Детектор — реальный, но **показывает** сигнал, а не решает; влияет на ~3–5 «развилочных» пиков из 42; **настроен и проверен на n=1 драфте** → пороги могут шуметь/молчать на других. Ожидаемый прирост винрейта живого драфта ≈ **+1–3%** ЕСЛИ дисциплинированно отрабатывать баннеры — в пределах дисперсии лимитеда тонет. **TODO: прогнать детектор на 5–10 следующих драфтах и подкрутить пороги по живым данным** (риск ложного пивота от пере-доверия баннеру). Если чёрный уже основной цвет, нужно сплэшнуть ТОЛЬКО зелёный: 3 зелёных источника (Terramorphic Expanse + 2 леса) — достаточно. Ошибка: видишь 2 цвета в мане → паникуешь «нужен суп».
